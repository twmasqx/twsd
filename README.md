# Nexus Vision (العالم الساحر)

ملف هذا المشروع يحتوي على نسخة مبدئية منظمة لواجهة رادار ومُحرّك فحص الشبكة.

المكونات الرئيسية:
- `setup.py`: سكربت لفحص وتثبيت الحزم الأساسية (PyQt6, scapy) والتحقق من وجود Npcap على ويندوز.
- `network_engine.py`: المحرك المسؤول عن فحص الشبكة باستخدام `scapy` مع `ARP` وL3 socket، وقاعدة OUI مبسطة للتعرّف على Apple, Samsung, Huawei, Xiaomi, Sony، ونظام محاكاة (fallback) عند عدم توفر الصلاحيات أو الحزم.
- `ui_core.py`: واجهة PyQt6 بحجم 420x850، تحتوي على `RadarWidget` مع أيقونات مرسومة عبر `QPainterPath`، ودوائر Glassmorphism، وأزرار سفلية تفتح شاشة المراقب.
- `main.py`: نقطة الدخول؛ تربط المحرك بالواجهة وتقوم بمسح مبدئي. لا تعيد تشغيل التطبيق تلقائياً بصلاحيات المسؤول (تُظهر تحذير بدلاً من ذلك).

أمر واحد لتثبيت كل المكتبات المطلوبة:

```bash
python -m pip install -r requirements.txt
```

ملاحظة حول Npcap على ويندوز:
- لتعمل فحوص الشبكة الحقيقية عبر `scapy` على ويندوز بشكل صحيح، يلزم تثبيت Npcap من الموقع الرسمي:
  https://nmap.org/npcap/

تشغيل التطبيق:

```bash
python main.py
```

ماذا فعلت هنا (ملخّص تقني):
- وضعت بنية مشروع منظمة بأربعة ملفات رئيسية كما طُلب.
- صممت `network_engine.py` ليتعامل مع عدم توافر `scapy` أو صلاحيات المسؤول عبر محاكاة أجهزة.
- بنية الواجهة في `ui_core.py` تستخدم `QPainterPath` للرسم البرمجي (لا صور خارجية) وميزات متحركة بسيطة.
- منعّت إعادة التشغيل التلقائي بصلاحيات المسؤول في `main.py` لتجنّب إغلاق التطبيق غير المتوقع؛ بدلاً من ذلك يُعرض تحذير.

إذا رغبت، أستطيع:
- إضافة لقطة شاشة أو تسجيل فيديو صغير للواجهة أثناء التشغيل.
- إنشاء ملف تثبيت Windows أو إعداد اختصار يطلق التطبيق بصلاحيات المسؤول عند الحاجة.
